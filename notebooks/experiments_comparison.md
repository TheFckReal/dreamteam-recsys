# Сравнение моделей рекомендательных систем

## 1. Обзор экспериментов

Исследовались три модели коллаборативной фильтрации для задачи рекомендаций:


| Модель                       | Источник                        | Тип                                      | Библиотека                     |
| -----------------------------| ------------------------------- | ---------------------------------------- | ------------------------------ |
| **Truncated SVD** (бейзлайн) | `svd_experiments_unified.ipynb` | Matrix Factorization (explicit-like)     | scikit-learn                   |
| **iALS** (implicit ALS)      | `ials_research.ipynb`           | Matrix Factorization (implicit feedback) | implicit                       |
| **EASE**                     | `ease_research.ipynb`           | Embarrassingly Shallow Autoencoder       | numpy (собственная реализация) |


### Данные

- **Датасет:** marketplace (T-ECD-small), ~130M событий, действия: `view`, `click`, `clickout`, `like`.
- **EASE** обучалась на ограниченном подмножестве (top-5000 популярных товаров) из-за нехватки оперативной памяти.

### Методология оценки

- **Разбиение:** Global Temporal Split 80/20 (по дням), 1 день-буфер между train и test.
- **Таргет (SVD, iALS):** взвешиванный таргет по обратной частоте по типам действий; проверялись варианты `target`, `log_target`, `sqrt_target`.
- **Таргет (EASE):** бинарный (факт взаимодействия = 1).
- **Метрики ранжирования:** NDCG@15, Precision@15, Recall@15.
- **Метрики регрессии (SVD, iALS):** RMSE, MAE.

## 5. Лучшее решение и выводы

### Лучшая модель: iALS (implicit ALS)

**Конфигурация:** `target_col=log_target, factors=16, regularization=0.133, alpha=9.04, iterations=18`

**Результат:** NDCG@15 = **0.1440** (на 6.1% выше SVD-бейзлайна)

### Почему iALS оказался лучшим

1. **Нативная работа с implicit feedback.** iALS (implicit Alternating Least Squares) изначально проектировался для данных об implicit обратной связи (просмотры, клики), в отличие от SVD, который предназначен для явных рейтингов. iALS вводит понятие весов уверенности (*confidence weighting*) — чем чаще пользователь взаимодействовал с товаром, тем выше уверенность модели в этом предпочтении (параметр $\alpha$). Это критически важно для данных маркетплейса, где большинство взаимодействий — просмотры (view).
2. **Низкоразмерное представление (factors=16).** Все лучшие конфигурации — и для SVD, и для iALS — используют малое число латентных факторов (16–20). При увеличении до 64–128 качество падает. Это говорит о том, что данные имеют относительно простую латентную структуру, и модели с большим числом параметров начинают переобучаться на шумных паттернах.
3. **Логарифмический таргет (`log_target`).** Использование inverse frequency weighting с логарифмическим сглаживанием стабильно даёт лучшие результаты по сравнению с `sqrt_target` и сырым `target`. Логарифм сжимает диапазон весов (view=0.71 vs like=8.05), предотвращая доминирование редких, но сверхвзвешенных событий (like: вес 3121 без логарифма).
4. **Эффективность обучения.** iALS обучается за ~1 минуту на полном датасете (1.2M users × 498K items) на GPU — в 30–60 раз быстрее SVD. Это позволило провести полноценную оптимизацию гиперпараметров через Optuna (25 trials), что было бы практически невозможно с SVD.
5. **Лучшие метрики регрессии.** RMSE (0.891 vs 3.111) и MAE (0.665 vs 0.995) у iALS значительно ниже, чем у SVD, что свидетельствует о более точном восстановлении значений таргета в матрице взаимодействий.

### О модели EASE

EASE показала самую высокую Precision@15 (0.0963) среди всех моделей, но значительно уступает по NDCG@15 и Recall@15. Это объясняется ограничением до top-5000 товаров из-за нехватки памяти — модель точно предсказывает из компактного набора, но не способна покрыть полное пространство предпочтений (~500K товаров). Преимущество EASE — простота реализации (замкнутая формула, без итеративного обучения).
